# Dark Mode Implementation - TeamFlow

## Przegląd
Implementacja dark mode w aplikacji TeamFlow z wykorzystaniem biblioteki next-themes.
Dark mode jest ustawiony jako domyślny motyw aplikacji.

## Zainstalowane biblioteki
- `next-themes@^0.4.6` - Zarządzanie motywami w Next.js

## Struktura implementacji

### 1. Konfiguracja główna
- **src/components/providers.tsx**: Dodano ThemeProvider z konfiguracją
  - `defaultTheme="dark"` - dark mode jako domyślny
  - `enableSystem` - obsługa motywu systemowego
  - `attribute="class"` - używa klas CSS do przełączania
  - `disableTransitionOnChange` - wyłącza animacje podczas zmiany

- **src/app/layout.tsx**: Dodano `suppressHydrationWarning` dla poprawnej obsługi SSR

### 2. CSS Variables
- **src/app/globals.css**: Wykorzystano istniejące zmienne CSS
  - `:root` - zmienne dla light mode
  - `.dark` - zmienne dla dark mode
  - Tailwind CSS skonfigurowany z `darkMode: ["class"]`

### 3. Komponenty UI

#### ThemeToggle (src/components/ui/theme-toggle.tsx)
- Dropdown menu z opcjami: Jasny, Ciemny, Systemowy
- Animowane ikony słońca i księżyca
- Używa useTheme hook z next-themes

#### TopBarUser (src/components/dashboard/top-bar-user.tsx)
- Dodano ThemeToggle obok menu użytkownika
- Import i użycie komponentu ThemeToggle

#### SettingsContent (src/components/settings/settings-content.tsx)
- Zaktualizowano sekcję "Wygląd" w ustawieniach
- Usunięto lokalny stan dla motywu
- Używa useTheme hook dla rzeczywistego przełączania
- Funkcjonalne przyciski: Jasny, Ciemny, Systemowy

### 4. Lokalizacje przełączników motywu

1. **Górny pasek nawigacji**:
   - Przycisk toggle obok menu użytkownika
   - Szybki dostęp z każdej strony aplikacji

2. **Ustawienia aplikacji**:
   - Zakładka "Wygląd"
   - Szczegółowe opcje konfiguracji motywu

## Funkcjonalności

### Obsługiwane motywy
- **Light** (Jasny) - jasny motyw
- **Dark** (Ciemny) - ciemny motyw (domyślny)
- **System** (Systemowy) - automatyczne dopasowanie do ustawień systemu

### Zachowanie
- Motyw jest zapisywany w localStorage
- Automatyczne przełączanie przy zmianie ustawień systemowych (tryb System)
- Płynne przejścia między motywami
- Brak migotania podczas ładowania strony (SSR)

### Dostępność
- Ikony z opisami dla screen readerów
- Keyboard navigation w dropdown menu
- Wysokie kontrasty w obu motywach

## Uwagi techniczne
- Używa CSS custom properties dla kolorów
- Kompatybilne z shadcn/ui components
- Wszystkie komponenty dashboardu zostały zaktualizowane do obsługi dark mode
- Zamieniono hardcoded kolory Tailwind (bg-white, text-gray-900, etc.) na CSS variables (bg-background, text-foreground, etc.)
- Funkcje kolorów dla priority i status używają teraz CSS variables z fallback dla dark mode
- Layout, sidebar, top bar i wszystkie główne komponenty obsługują dark mode
