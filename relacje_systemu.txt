RELACJE MIĘDZY ELEMENTAMI SYSTEMU TEAMFLOW
=============================================

## GŁÓWNE MODELE I ICH RELACJE

### 1. USER (Użytkownik)
- **Relacje wychodzące:**
  - Należy do wielu TEAM (many-to-many)
  - Może być przypisany do wielu TASK jako assignee (one-to-many)
  - Może tworzyć TASK jako creator (one-to-many)
  - Może blokować TASK jako blocker (one-to-many)
  - Tworzy wpisy TIME_ENTRY (one-to-many)
  - Tworzy COMMENT (one-to-many)
  - Uploaduje PROJECT_DOCUMENT (one-to-many)
  - Uploaduje TASK_ATTACHMENT (one-to-many)
  - Tworzy SYSTEM_CHANGE (one-to-many)
  - Czyta SYSTEM_CHANGE_READ (one-to-many)
  - Ma PUSH_SUBSCRIPTION (one-to-many)
  - Tworzy CHAT_ROOM (one-to-many)
  - Wysyła MESSAGE (one-to-many)
  - Należy do USER_CHAT_ROOM (one-to-many)
  - Ma PASSWORD_RESET_TOKEN (one-to-many)

### 2. TEAM (Zespół)
- **Relacje wychodzące:**
  - Ma wielu członków USER (many-to-many)
  - Ma wiele PROJECT (one-to-many)

### 3. PROJECT (Projekt)
- **Relacje wychodzące:**
  - Należy do jednego TEAM (many-to-one)
  - Ma wiele TASK (one-to-many)
  - Ma wiele PROJECT_DOCUMENT (one-to-many)
  - Ma wiele CHAT_ROOM (one-to-many)

- **Pola specjalne:**
  - archived: Boolean - określa czy projekt jest zarchiwizowany
  - imageUrl: String - ścieżka do obrazu projektu
  - shareToken: String - token do publicznego udostępniania

### 4. TASK (Zadanie)
- **Relacje wychodzące:**
  - Należy do jednego PROJECT (many-to-one, opcjonalne)
  - Ma jednego assignee USER (many-to-one, opcjonalne)
  - Ma jednego creator USER (many-to-one, opcjonalne)
  - Ma jednego blocker USER (many-to-one, opcjonalne)
  - Ma jeden TASK_STATUS (many-to-one, opcjonalne)
  - Ma wiele COMMENT (one-to-many)
  - Ma wiele SUBTASK (one-to-many)
  - Ma wiele TASK_IMAGE (one-to-many)
  - Ma wiele TASK_ATTACHMENT (one-to-many)
  - Ma wiele TIME_ENTRY (one-to-many)
  - Ma wiele TODO (one-to-many)

### 5. TASK_STATUS (Status zadania)
- **Relacje wychodzące:**
  - Ma wiele TASK (one-to-many)
- **Uwaga:** Statusy są globalne, nie per projekt

### 6. SUBTASK (Podzadanie)
- **Relacje wychodzące:**
  - Należy do jednego TASK (many-to-one)

### 7. TODO (Lista zadań do wykonania)
- **Relacje wychodzące:**
  - Należy do jednego TASK (many-to-one)
- **Cascade delete:** Tak - usuwane z TASK

### 8. COMMENT (Komentarz)
- **Relacje wychodzące:**
  - Należy do jednego TASK (many-to-one)
  - Ma jednego autora USER (many-to-one)
- **Cascade delete:** Tak - usuwane z TASK i USER

### 9. TIME_ENTRY (Wpis czasu pracy)
- **Relacje wychodzące:**
  - Należy do jednego TASK (many-to-one)
  - Należy do jednego USER (many-to-one)

### 10. PROJECT_DOCUMENT (Dokument projektu)
- **Relacje wychodzące:**
  - Należy do jednego PROJECT (many-to-one)
  - Ma jednego uplodera USER (many-to-one)
- **Cascade delete:** Tak - usuwane z PROJECT
- **Pliki:** Przechowywane w /uploads/projects/{projectId}/documents/

### 11. TASK_IMAGE (Obraz zadania)
- **Relacje wychodzące:**
  - Należy do jednego TASK (many-to-one)
- **Cascade delete:** Tak - usuwane z TASK
- **Pliki:** Przechowywane w /uploads/tasks/{taskId}/

### 12. TASK_ATTACHMENT (Załącznik zadania)
- **Relacje wychodzące:**
  - Należy do jednego TASK (many-to-one)
  - Ma jednego uplodera USER (many-to-one)
- **Cascade delete:** Tak - usuwane z TASK i USER
- **Pliki:** Przechowywane w /uploads/tasks/{taskId}/attachments/

### 13. CHAT_ROOM (Pokój czatu)
- **Relacje wychodzące:**
  - Może należeć do jednego PROJECT (many-to-one, opcjonalne)
  - Ma jednego twórcy USER (many-to-one)
  - Ma wiele MESSAGE (one-to-many)
  - Ma wiele USER_CHAT_ROOM (one-to-many)
- **Cascade delete:** Tak - usuwane z PROJECT

### 14. MESSAGE (Wiadomość)
- **Relacje wychodzące:**
  - Należy do jednego CHAT_ROOM (many-to-one)
  - Ma jednego nadawcę USER (many-to-one)
- **Cascade delete:** Tak - usuwane z CHAT_ROOM

### 15. USER_CHAT_ROOM (Członkostwo w pokoju czatu)
- **Relacje wychodzące:**
  - Należy do jednego USER (many-to-one)
  - Należy do jednego CHAT_ROOM (many-to-one)
- **Cascade delete:** Tak - usuwane z USER i CHAT_ROOM

## USUWANIE PROJEKTÓW - KASKADOWE USUWANIE

### Automatyczne usuwanie przez Prisma (onDelete: Cascade):
1. **PROJECT_DOCUMENT** - usuwane automatycznie
2. **CHAT_ROOM** - usuwane automatycznie
3. **MESSAGE** (przez CHAT_ROOM) - usuwane automatycznie
4. **USER_CHAT_ROOM** (przez CHAT_ROOM) - usuwane automatycznie

### Usuwanie przez relacje TASK:
1. **TASK** - usuwane automatycznie (brak explicit cascade, ale działa)
2. **COMMENT** (przez TASK) - usuwane automatycznie (onDelete: Cascade)
3. **SUBTASK** (przez TASK) - usuwane automatycznie (brak explicit cascade)
4. **TODO** (przez TASK) - usuwane automatycznie (onDelete: Cascade)
5. **TASK_IMAGE** (przez TASK) - usuwane automatycznie (onDelete: Cascade)
6. **TASK_ATTACHMENT** (przez TASK) - usuwane automatycznie (onDelete: Cascade)
7. **TIME_ENTRY** (przez TASK) - usuwane automatycznie (brak explicit cascade)

### Pliki wymagające ręcznego usunięcia:
1. **Obraz projektu:** /uploads/projects/{projectId}_*.{ext}
2. **Dokumenty projektu:** /uploads/projects/{projectId}/documents/
3. **Obrazy zadań:** /uploads/tasks/{taskId}/
4. **Załączniki zadań:** /uploads/tasks/{taskId}/attachments/
5. **Katalogi projektów:** /uploads/projects/{projectId}/

## STRUKTURA KATALOGÓW PLIKÓW

```
public/uploads/
├── avatars/           # Awatary użytkowników
├── projects/          # Pliki projektów
│   ├── {projectId}_timestamp.ext    # Obrazy projektów
│   └── {projectId}/
│       └── documents/ # Dokumenty projektów
└── tasks/             # Pliki zadań
    └── {taskId}/
        ├── timestamp.ext           # Obrazy zadań
        └── attachments/           # Załączniki zadań
```

## BEZPIECZEŃSTWO USUWANIA

### Warunki usuwania projektu:
1. Użytkownik musi być członkiem zespołu projektu LUB być administratorem
2. Projekt musi być zarchiwizowany (archived = true)
3. Potwierdzenie przez dialog AlertDialog

### Proces usuwania:
1. Pobranie projektu z wszystkimi powiązanymi danymi
2. Zebranie ścieżek do wszystkich plików
3. Usunięcie plików z systemu plików
4. Usunięcie katalogów
5. Usunięcie projektu z bazy danych (kaskadowe usuwanie reszty)

### Obsługa błędów:
- Błędy usuwania plików są logowane jako ostrzeżenia
- Usuwanie z bazy danych następuje niezależnie od powodzenia usuwania plików
- Użytkownik otrzymuje informację o liczbie usuniętych plików i katalogów
