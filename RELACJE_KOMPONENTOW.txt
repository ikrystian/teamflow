RELACJE MIĘDZY KOMPONENTAMI - GLOBALNE STATUSY ZADAŃ

=== GŁÓWNE KOMPONENTY ===

1. KanbanBoard (src/components/projects/kanban-board.tsx)
   - Główny komponent tablicy zadań dla projektów
   - Zarządza statusami zadań (kolumnami) - teraz pobiera globalne statusy
   - Zawiera przycisk "Dodaj kolumnę" na końcu listy kolumn
   - Używa TaskStatusDialog do tworzenia nowych statusów (przekierowuje do globalnego API)
   - FUNKCJONALNOŚĆ: Każda kolumna ma QuickAddTask do szybkiego dodawania zadań

2. TasksKanbanBoard (src/components/tasks/tasks-kanban-board.tsx)
   - Główny komponent tablicy zadań dla widoku "Wszystkie zadania"
   - ZAKTUALIZOWANY: Teraz pobiera globalne statusy z API (/api/system/task-statuses)
   - Używa tych samych statusów co są zdefiniowane w ustawieniach
   - Obsługuje drag & drop z aktualizacją statusId i status name
   - FUNKCJONALNOŚĆ: Każda kolumna ma QuickAddTask do szybkiego dodawania zadań

3. SystemTaskStatuses (src/components/settings/system-task-statuses.tsx)
   - NOWY KOMPONENT: Zarządzanie globalnymi statusami zadań w ustawieniach systemowych
   - Pozwala na tworzenie, edycję, usuwanie i zmianę kolejności statusów
   - Używa SystemTaskStatusDialog do edycji statusów
   - Drag & drop do zmiany kolejności

4. SystemTaskStatusDialog (src/components/settings/system-task-status-dialog.tsx)
   - NOWY KOMPONENT: Dialog do tworzenia/edycji globalnych statusów zadań
   - Pozwala na ustawienie nazwy, koloru i czy status jest domyślny
   - Używany w ustawieniach systemowych

5. TaskStatusDialog (src/components/projects/task-status-dialog.tsx)
   - Dialog do tworzenia/edycji statusów zadań (zachowany dla kompatybilności)
   - Teraz przekierowuje do globalnego API zamiast API per-projekt

6. KanbanColumn (wewnętrzny komponent w obu kanban-board.tsx)
   - Reprezentuje pojedynczą kolumnę w tablicy
   - Zawiera zadania dla danego statusu
   - Obsługuje drag & drop zadań
   - FUNKCJONALNOŚĆ: Zawiera QuickAddTask na dole kolumny

7. QuickAddTask (wewnętrzny komponent w obu kanban-board.tsx)
   - Pozwala na szybkie dodawanie zadań bezpośrednio w kolumnie
   - Automatycznie ustawia status zadania na podstawie kolumny (używa statusId)
   - Używa prostego formularza z tytułem zadania
   - Automatycznie przypisuje zadanie do aktualnego użytkownika

=== PRZEPŁYW DANYCH ===

Ustawienia systemowe:
├── SystemTaskStatuses pobiera globalne statusy z API (/api/system/task-statuses)
├── Renderuje listę statusów z możliwością edycji
├── Używa SystemTaskStatusDialog do tworzenia/edycji
└── Drag & drop do zmiany kolejności

KanbanBoard (projekty):
├── Pobiera globalne statusy zadań z API (/api/projects/[projectId]/task-statuses)
├── API przekierowuje do globalnych statusów
├── Renderuje kolumny dla każdego statusu
├── Dodaje przycisk "Dodaj kolumnę" na końcu
└── Otwiera TaskStatusDialog po kliknięciu przycisku (przekierowuje do globalnego API)

TasksKanbanBoard (wszystkie zadania):
├── Pobiera globalne statusy bezpośrednio z API (/api/system/task-statuses)
├── Renderuje kolumny dla każdego statusu z globalnych ustawień
├── Obsługuje drag & drop z aktualizacją statusId i status name
├── Filtruje zadania według statusId (jeśli dostępne) lub status name
└── QuickAddTask używa statusId do tworzenia zadań

=== API ENDPOINTS ===

GLOBALNE STATUSY:
GET /api/system/task-statuses
- Pobiera wszystkie globalne statusy zadań
- Tworzy domyślne statusy jeśli nie istnieją

POST /api/system/task-statuses
- Tworzy nowy globalny status zadania
- Wymaga: name, color, isDefault
- Automatycznie ustawia kolejność (order)

PATCH /api/system/task-statuses/[statusId]
- Aktualizuje globalny status zadania

DELETE /api/system/task-statuses/[statusId]
- Usuwa globalny status zadania (tylko jeśli nie jest używany)

POST /api/system/task-statuses/reorder
- Zmienia kolejność globalnych statusów

KOMPATYBILNOŚĆ Z PROJEKTAMI:
GET /api/projects/[projectId]/task-statuses
- Przekierowuje do globalnych statusów
- Zachowana kompatybilność z istniejącymi komponentami

POST /api/projects/[projectId]/task-statuses
- Przekierowuje do globalnego API tworzenia statusów

=== STRUKTURA BAZY DANYCH ===

TaskStatus (ZMIENIONA - GLOBALNA):
- id: string (cuid)
- name: string (nazwa statusu)
- color: string (kolor hex)
- order: int (kolejność w tablicy)
- isDefault: boolean (czy domyślny dla nowych zadań)
- USUNIĘTE: projectId (statusy są teraz globalne)

=== FUNKCJONALNOŚCI ===

1. Zarządzanie globalnymi statusami (Ustawienia systemowe):
   - Nowa zakładka "Statusy zadań" w ustawieniach systemowych
   - Tworzenie, edycja, usuwanie globalnych statusów
   - Drag & drop do zmiany kolejności
   - Wybór koloru z palety predefiniowanych kolorów
   - Ustawianie statusu jako domyślny

2. Dodawanie kolumny z tablicy (zachowane):
   - Przycisk "Dodaj kolumnę" na końcu tablicy
   - Otwiera dialog TaskStatusDialog
   - Teraz tworzy globalne statusy zamiast per-projekt
   - Po zapisaniu automatycznie odświeża tablicę

3. Integracja z istniejącymi funkcjami:
   - Drag & drop zadań między kolumnami
   - Automatyczne przypisywanie statusu do zadań
   - Wszystkie projekty używają tych samych statusów
   - Zachowana kompatybilność z istniejącymi komponentami

- Hover efekty
- Ikona Plus z tekstem
- Szerokość 72 (w-72) jak inne kolumny
- Wysokość dopasowana do zawartości (h-fit)

Dialog:
- Wykorzystuje shadcn/ui komponenty

=== BEZPIECZEŃSTWO ===

- Weryfikacja uprawnień użytkownika (tylko zalogowani użytkownicy)
- Walidacja danych wejściowych
- Unikalne nazwy statusów globalnie
- Unikalna kolejność statusów globalnie
- Sprawdzanie czy status nie jest używany przed usunięciem

=== MIGRACJA Z STATUSÓW PER-PROJEKT DO GLOBALNYCH ===

Zmiany w bazie danych:
- Usunięto pole projectId z modelu TaskStatus
- Usunięto ograniczenia unique per projekt
- Dodano globalne ograniczenia unique
- Zachowano relację z zadaniami

Kompatybilność wsteczna:
- API projektów przekierowuje do globalnych statusów
- Istniejące komponenty działają bez zmian
- Zachowane endpointy dla kompatybilności

=== FUNKCJONALNOŚĆ - AUTOMATYCZNE USTAWIANIE STATUSU ===

QuickAddTask komponent:
- Dodany do każdej kolumny w kanban board
- Automatycznie ustawia status zadania na podstawie kolumny
- Prosty formularz z tytułem zadania
- Eliminuje potrzebę ręcznego wybierania statusu

Przepływ:
1. Użytkownik klika "Dodaj zadanie" w konkretnej kolumnie
2. Pojawia się formularz z polem tytułu
3. Po wysłaniu zadanie jest tworzone z automatycznym statusem
4. Zadanie pojawia się w odpowiedniej kolumnie
