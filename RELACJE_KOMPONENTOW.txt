=== API ENDPOINTS ===

GLOBALNE STATUSY:
GET /api/system/task-statuses
- Pobiera wszystkie globalne statusy zadań
- Tworzy domyślne statusy jeśli nie istnieją

POST /api/system/task-statuses
- Tworzy nowy globalny status zadania
- Wymaga: name, color, isDefault
- Automatycznie ustawia kolejność (order)

PATCH /api/system/task-statuses/[statusId]
- Aktualizuje globalny status zadania

DELETE /api/system/task-statuses/[statusId]
- Usuwa globalny status zadania (tylko jeśli nie jest używany)

POST /api/system/task-statuses/reorder
- Zmienia kolejność globalnych statusów

KOMPATYBILNOŚĆ Z PROJEKTAMI:
GET /api/projects/[projectId]/task-statuses
- Przekierowuje do globalnych statusów
- Zachowana kompatybilność z istniejącymi komponentami

POST /api/projects/[projectId]/task-statuses
- USUNIĘTO: Endpoint został usunięty wraz z funkcjonalnością dodawania kolumn
