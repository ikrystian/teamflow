TeamFlow - Relacje między elementami systemu

## Struktura bazy danych (Prisma Schema)

### User (Użytkownik)
- id: String (PK)
- name: String?
- email: String (unique)
- password: String
- avatarUrl: String?
- createdAt: DateTime
- updatedAt: DateTime

Relacje:
- teams: Team[] (many-to-many przez "TeamMembers")
- assignedTasks: Task[] (one-to-many) - zadania przypisane do użytkownika
- createdTasks: Task[] (one-to-many) - zadania utworzone przez użytkownika
- comments: Comment[] (one-to-many)
- timeEntries: TimeEntry[] (one-to-many) - wpisy czasu pracy

### Team (Zespół)
- id: String (PK)
- name: String
- createdAt: DateTime
- updatedAt: DateTime

Relacje:
- members: User[] (many-to-many przez "TeamMembers")
- projects: Project[] (one-to-many)

### Project (Projekt)
- id: String (PK)
- name: String
- description: String?
- status: String (default: "In Progress")
- createdAt: DateTime
- updatedAt: DateTime
- teamId: String (FK)

Relacje:
- team: Team (many-to-one)
- tasks: Task[] (one-to-many)
- taskStatuses: TaskStatus[] (one-to-many) - konfigurowalne statusy zadań

### Task (Zadanie)
- id: String (PK)
- title: String
- description: String?
- status: String (default: "To Do") - przestarzałe, zastąpione przez statusId
- statusId: String? (FK) - referencja do TaskStatus
- priority: String?
- dueDate: DateTime?
- estimatedHours: Float? (szacowany czas w godzinach)
- createdAt: DateTime
- updatedAt: DateTime
- projectId: String (FK)
- assigneeId: String? (FK)
- createdById: String (FK) - autor zadania

Relacje:
- project: Project (many-to-one)
- taskStatus: TaskStatus? (many-to-one) - konfigurowalny status
- assignee: User? (many-to-one)
- createdBy: User (many-to-one) - autor zadania
- subtasks: Subtask[] (one-to-many)
- comments: Comment[] (one-to-many)
- timeEntries: TimeEntry[] (one-to-many) - wpisy czasu pracy
- images: TaskImage[] (one-to-many) - obrazki załączone do zadania

### TaskStatus (Status Zadania)
- id: String (PK)
- name: String - nazwa statusu (np. "To Do", "In Progress", "Done", "Review")
- color: String (default: "#6B7280") - kolor hex dla statusu
- order: Int - kolejność statusu na tablicy (0, 1, 2, etc.)
- isDefault: Boolean (default: false) - czy to domyślny status dla nowych zadań
- createdAt: DateTime
- updatedAt: DateTime
- projectId: String (FK)

Relacje:
- project: Project (many-to-one)
- tasks: Task[] (one-to-many) - zadania z tym statusem

Ograniczenia:
- Unikalna nazwa statusu per projekt
- Unikalna kolejność per projekt

Uwagi:
- Przy tworzeniu nowego projektu automatycznie tworzone są domyślne statusy: "To Do" (domyślny), "In Progress", "Done"
- Domyślne statusy są zapisywane w bazie danych, nie są już zwracane jako tymczasowe dane
- Dla istniejących projektów bez statusów, domyślne statusy są tworzone przy pierwszym pobraniu

### Subtask (Podzadanie)
- id: String (PK)
- title: String
- isCompleted: Boolean (default: false)
- createdAt: DateTime
- updatedAt: DateTime
- taskId: String (FK)

Relacje:
- task: Task (many-to-one)

### Comment (Komentarz)
- id: String (PK)
- content: String
- createdAt: DateTime
- updatedAt: DateTime
- taskId: String (FK)
- authorId: String (FK)

Relacje:
- task: Task (many-to-one)
- author: User (many-to-one)

### TimeEntry (Wpis czasu pracy)
- id: String (PK)
- hours: Float (czas w godzinach, np. 0.5 = 30 minut)
- description: String? (opcjonalny opis pracy)
- date: DateTime (data wykonania pracy)
- createdAt: DateTime
- updatedAt: DateTime
- taskId: String (FK)
- userId: String (FK)

Relacje:
- task: Task (many-to-one)
- user: User (many-to-one)

### TaskImage (Obrazek zadania)
- id: String (PK)
- filename: String (oryginalna nazwa pliku)
- url: String (ścieżka do zapisanego obrazka)
- mimeType: String (typ MIME obrazka)
- size: Int (rozmiar pliku w bajtach)
- createdAt: DateTime
- taskId: String (FK)

Relacje:
- task: Task (many-to-one) - z kaskadowym usuwaniem

Ograniczenia:
- Maksymalny rozmiar pliku: 5MB
- Obsługiwane formaty: wszystkie typy image/*
- Pliki są zapisywane w public/uploads/tasks/{taskId}/

## API Endpoints

### Authentication
- POST /api/auth/register - Rejestracja użytkownika
- GET/POST /api/auth/[...nextauth] - NextAuth.js handler

### Teams
- GET /api/teams - Lista zespołów użytkownika
- POST /api/teams - Tworzenie nowego zespołu
- GET /api/teams/[teamId] - Szczegóły pojedynczego zespołu
- PATCH /api/teams/[teamId] - Edycja zespołu (nazwa i członkowie)
- GET /api/teams/[teamId]/members - Lista członków zespołu
- POST /api/teams/[teamId]/members - Dodawanie członka do zespołu
- DELETE /api/teams/[teamId]/members?userId=X - Usuwanie członka z zespołu

### Users
- GET /api/users - Lista wszystkich użytkowników (z opcjonalnym wyszukiwaniem)
- GET /api/user/profile - Pobieranie profilu aktualnego użytkownika
- PUT /api/user/profile - Aktualizacja profilu użytkownika
- POST /api/user/avatar - Przesyłanie zdjęcia profilowego
- DELETE /api/user/avatar - Usuwanie zdjęcia profilowego

### Projects
- GET /api/projects - Lista projektów (z filtrowaniem po teamId)
- POST /api/projects - Tworzenie nowego projektu
- GET /api/projects/[projectId] - Szczegóły pojedynczego projektu (BRAKUJE - powoduje 404)
- GET /api/projects/[projectId]/members - Lista członków projektu

### Tasks
- GET /api/tasks - Lista zadań (z filtrowaniem po projectId, assigneeId)
- POST /api/tasks - Tworzenie nowego zadania (z szacowanym czasem)
- GET /api/tasks/[taskId] - Szczegóły pojedynczego zadania
- PATCH /api/tasks/[taskId] - Edycja zadania (tylko autor lub przypisana osoba)

### Task Images (Obrazki zadań)
- POST /api/tasks/[taskId]/images - Przesyłanie obrazka do zadania
- DELETE /api/tasks/[taskId]/images?imageId=X - Usuwanie obrazka z zadania

### Time Tracking
- GET /api/tasks/[taskId]/time-entries - Lista wpisów czasu dla zadania
- POST /api/tasks/[taskId]/time-entries - Dodawanie wpisu czasu
- DELETE /api/tasks/[taskId]/time-entries?entryId=X - Usuwanie wpisu czasu

### Reports (Raporty)
- GET /api/reports/time-tracking - Raport czasu pracy z filtrami (startDate, endDate, projectId, userId, teamId)
- GET /api/reports/project-progress - Raport postępu projektów z filtrami (startDate, endDate, projectId, teamId)
- POST /api/reports/export - Eksport raportów do PDF/Excel/CSV (reportType, format, filtry)

## Przepływ danych

1. Użytkownik loguje się przez NextAuth.js
2. Session zawiera user.id używane w API calls
3. API routes sprawdzają autoryzację przez getServerSession
4. Prisma ORM wykonuje zapytania do SQLite
5. Komponenty React pobierają dane przez fetch() do API endpoints
6. Stan aplikacji zarządzany przez useState/useEffect

## Bezpieczeństwo

- Wszystkie API routes wymagają autoryzacji
- Użytkownicy widzą tylko swoje zespoły/projekty/zadania
- Hasła hashowane przez bcryptjs
- Session zarządzane przez NextAuth.js JWT
