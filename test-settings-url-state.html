🔧 Test URL State w Panelu Ustawień

📋 Instrukcje testowania

Ten test sprawdza czy panel ustawień poprawnie zapisuje stan zakładek w URL. Kliknij w linki poniżej, aby przetestować różne scenariusze.

Aplikacja musi być uruchomiona na: http://localhost:3001

🔗 Test 1: Bezpośrednie linki do zakładek

Kliknij w linki poniżej. Każdy powinien otworzyć odpowiednią zakładkę w ustawieniach:

👤 Profil 🔐 Bezpieczeństwo 🔔 Powiadomienia 🛡️ Prywatność 🎨 Wygląd ⚙️ Statusy zadań

🔴 Linki tylko dla administratorów:

📧 SMTP (Admin) 👥 Użytkownicy (Admin)

⚠️ Test 2: Walidacja nieprawidłowych parametrów

Te linki powinny przekierować do zakładki "Profil":

❌ Nieprawidłowa zakładka 🔄 Pusty parametr 🚫 Nieistniejąca zakładka

🧪 Test 3: Domyślne zachowanie

Ten link powinien otworzyć zakładkę "Profil" (domyślną):

🏠 Ustawienia (bez parametru)

✅ Lista kontrolna testów

Po kliknięciu w każdy link sprawdź:

  • Czy otwiera się odpowiednia zakładka w ustawieniach
  • Czy URL w pasku adresu zawiera prawidłowy parametr ?tab=
  • Czy zakładka jest wizualnie aktywna (podświetlona)
  • Czy zawartość zakładki się wyświetla

Test nawigacji w przeglądarce:

  • Przejdź przez kilka zakładek klikając w nie
  • Użyj przycisku "Wstecz" w przeglądarce
  • Sprawdź czy wraca do poprzedniej zakładki
  • Użyj przycisku "Wprzód" w przeglądarce

Test odświeżenia strony:

  • Przejdź do dowolnej zakładki (np. Powiadomienia)
  • Odśwież stronę (F5 lub Ctrl+R)
  • Sprawdź czy pozostajesz w tej samej zakładce
  • Sprawdź czy URL się nie zmienił

Test uprawnień (jeśli nie jesteś adminem):

  • Kliknij link "SMTP (Admin)"
  • Sprawdź czy przekierowuje do zakładki "Profil"
  • Kliknij link "Użytkownicy (Admin)"
  • Sprawdź czy przekierowuje do zakładki "Profil"

🔍 Debugowanie

Jeśli coś nie działa:

  • Sprawdź console przeglądarki (F12) - czy są błędy JavaScript
  • Sprawdź Network tab - czy są błędy 404 lub 500
  • Sprawdź czy jesteś zalogowany - ustawienia wymagają uwierzytelnienia
  • Sprawdź URL - czy zawiera prawidłowy parametr ?tab=

Oczekiwane zachowania:

  • Prawidłowe zakładki otwierają się bezpośrednio
  • URL aktualizuje się przy zmianie zakładki
  • Nieprawidłowe parametry przekierowują do "Profil"
  • Zakładki admin-only są niedostępne dla zwykłych użytkowników

📊 Wyniki testów

Po przeprowadzeniu wszystkich testów, URL state management w panelu ustawień powinien:

  • ✅ Zapisywać aktualną zakładkę w URL
  • ✅ Pozwalać na bezpośrednie linkowanie do zakładek
  • ✅ Zachowywać stan po odświeżeniu strony
  • ✅ Obsługiwać nawigację wstecz/wprzód
  • ✅ Walidować uprawnienia użytkownika
  • ✅ Przekierowywać nieprawidłowe parametry
