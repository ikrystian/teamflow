# Relacje między elementami systemu zarządzania zadaniami

## Struktura bazy danych

### Task (Zadanie)
- **id**: String (PK)
- **title**: String - tytuł zadania
- **description**: String? - opis zadania
- **status**: String - status zadania (domyślnie "To Do")
- **priority**: String? - priorytet zadania (High, Medium, Low)
- **dueDate**: DateTime? - termin wykonania
- **estimatedHours**: Float? - szacowany czas w godzinach
- **projectId**: String (FK) - odniesienie do projektu
- **assigneeId**: String? (FK) - przypisana osoba
- **createdById**: String (FK) - autor zadania
- **statusId**: String? (FK) - odniesienie do niestandardowego statusu

### Relacje Task:
- **project**: Project (Many-to-One) - zadanie należy do jednego projektu
- **assignee**: User? (Many-to-One) - zadanie może być przypisane do użytkownika
- **createdBy**: User (Many-to-One) - zadanie ma jednego autora
- **subtasks**: Subtask[] (One-to-Many) - zadanie może mieć podzadania
- **comments**: Comment[] (One-to-Many) - zadanie może mieć komentarze
- **timeEntries**: TimeEntry[] (One-to-Many) - zadanie może mieć wpisy czasu
- **images**: TaskImage[] (One-to-Many) - zadanie może mieć załączone obrazy
- **todos**: Todo[] (One-to-Many) - zadanie może mieć listę do zrobienia

## API Endpoints

### /api/tasks
- **GET**: Pobiera listę zadań (z filtrowaniem)
- **POST**: Tworzy nowe zadanie

### /api/tasks/[taskId]
- **GET**: Pobiera szczegóły konkretnego zadania
- **PATCH**: Aktualizuje zadanie
- **DELETE**: Usuwa zadanie z kaskadowym usuwaniem powiązanych danych

### /api/tasks/[taskId]/time-entries
- **GET**: Pobiera wpisy czasu dla zadania
- **POST**: Dodaje nowy wpis czasu
- **DELETE**: Usuwa wpis czasu

### /api/tasks/[taskId]/images
- **POST**: Dodaje obraz do zadania
- **DELETE**: Usuwa obraz z zadania

## Uprawnienia

### Kto może usunąć zadanie:
1. **Autor zadania** (createdBy) - zawsze może usunąć
2. **Przypisana osoba** (assignee) - może usunąć zadanie przypisane do siebie
3. **Członek zespołu** - może usunąć zadania w projektach swojego zespołu (sprawdzenie przez project.team.members)

### Sprawdzenia bezpieczeństwa:
- Weryfikacja sesji użytkownika
- Sprawdzenie czy zadanie istnieje
- Sprawdzenie uprawnień do usunięcia
- Sprawdzenie czy użytkownik ma dostęp do projektu

## Kaskadowe usuwanie

### Przy usunięciu zadania należy usunąć:
1. **Subtasks** - wszystkie podzadania
2. **Comments** - wszystkie komentarze
3. **TimeEntries** - wszystkie wpisy czasu
4. **TaskImages** - wszystkie obrazy (pliki z dysku + rekordy z bazy)

### Konfiguracja Prisma:
- Relacje powinny mieć onDelete: Cascade dla automatycznego usuwania powiązanych rekordów

## Poprawki i rozwiązane problemy

### Problem z niewidocznymi todos przy pierwszym otwarciu okna szczegółów zadania

**Data**: 2025-01-17
**Problem**: Przy pierwszym otworzeniu okienka ze szczegółami taska nie były widoczne zapisane todos.

**Przyczyna**:
- Komponent TaskTodos używał useState(initialTodos) do inicjalizacji stanu
- Nie było synchronizacji między props a stanem lokalnym
- Gdy TaskDetailsDialog pobierał aktualne dane z API i aktualizował props todos, komponent TaskTodos nie aktualizował swojego stanu

**Rozwiązanie**:
- Dodano useEffect w TaskTodos do synchronizacji stanu z props
- useEffect reaguje na zmiany w props initialTodos i aktualizuje lokalny stan todos

**Zmienione pliki**:
- `src/components/tasks/task-todos.tsx` - dodano useEffect do synchronizacji stanu

**Kod rozwiązania**:
```typescript
// Sync internal state when todos prop changes
useEffect(() => {
  setTodos(initialTodos)
}, [initialTodos])
```

**Relacja komponentów**:
TaskDetailsDialog → (pobiera dane z API) → TaskTodos (synchronizuje stan z props)
