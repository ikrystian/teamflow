# Relacje między elementami w TeamFlow

## Funkcjonalność blokowania zadań (2025-01-30):

### TaskBlockDialog (src/components/tasks/task-block-dialog.tsx)
- **Cel**: Dialog do blokowania/odblokowywania zadań z formularzem powodu blokady
- **Funkcjonalności**:
  - Formularz z polem tekstowym do opisu powodu blokady
  - Walidacja uprawnień (tylko autor lub przypisana osoba może blokować/odblokować)
  - Wyświetlanie informacji o zablokowanym zadaniu (powód, kto zablokował)
  - Potwierdzenie odblokowywania zadania
  - Toast notifications dla sukcesu/błędów
  - Obsługa stanów ładowania

### Rozszerzenia modelu Task:
- **isBlocked**: Boolean - czy zadanie jest zablokowane
- **blockReason**: String - powód blokady
- **blockedAt**: DateTime - kiedy zadanie zostało zablokowane
- **blockedById**: String - ID użytkownika który zablokował zadanie
- **blockedBy**: User relation - relacja do użytkownika który zablokował

### API endpoints:
- **POST /api/tasks/[taskId]/block** - blokowanie zadania z powodem
- **DELETE /api/tasks/[taskId]/block** - odblokowywanie zadania
- Aktualizacja istniejących endpoints o include blockedBy

### Wizualne oznaczenia zablokowanych zadań:
- **Czerwone obramowanie** i tło (bg-red-50/50, border-red-500) we wszystkich widokach
- **Ikona blokady** (Lock) obok tytułu zadania
- **Czerwony kolor** paska projektu w tabeli zadań
- Oznaczenia w: TasksKanbanBoard, KanbanBoard (projekty), TasksContent (lista), TasksTable (dashboard)

### Funkcje pomocnicze (src/lib/task-utils.ts):
- **isTaskBlocked()** - sprawdza czy zadanie jest zablokowane
- **canUserBlockTask()** - sprawdza czy użytkownik może blokować/odblokować zadanie
- **getBlockedTaskStyles()** - zwraca style dla zablokowanych zadań

### Integracja z istniejącymi komponentami:
- **TasksKanbanBoard**: dodano opcję blokowania w menu dropdown kart zadań
- **TasksContent**: dodano obsługę dialogu blokowania i przekazywanie do TasksKanbanBoard
- **TasksTable**: dodano ikonę blokady i czerwone oznaczenie w kolumnie nazwy
- **KanbanBoard** (projekty): dodano wizualne oznaczenia zablokowanych zadań

## AUTOMATYCZNE PRZYPISYWANIE ZADAŃ Z WIDOKU "MOJE ZADANIA"

### Lokalizacja: src/components/shared/task-form-dialog.tsx

**Problem:** Gdy użytkownik otwierał okienko "Utwórz nowe zadanie" z widoku "Moje zadania", po wybraniu projektu mógł zmienić przypisaną osobę na kogoś innego, co było nieintuicyjne.

**Rozwiązanie:** Dodano parametr `forceAssignToCurrentUser` do TaskFormDialog:

**Zmiany w interfejsie:**
```typescript
interface TaskFormDialogProps {
  // ... inne właściwości
  forceAssignToCurrentUser?: boolean // When true, always assign to current user regardless of project
}
```

**Logika działania:**
1. Gdy `forceAssignToCurrentUser=true`, pole przypisanej osoby jest zablokowane
2. Zawsze pokazuje aktualnego użytkownika jako przypisanego
3. Nie pozwala na zmianę przypisania niezależnie od wybranego projektu
4. Automatycznie ustawia assigneeId na session.user.id przy zmianie projektu

**Implementacja w tasks-content.tsx:**
```typescript
<TaskFormDialog
  mode="create"
  // ... inne props
  forceAssignToCurrentUser={filter === "assigned"}
/>
```

**Efekt:** Gdy użytkownik jest w widoku "Moje zadania" (filter === "assigned"), nowe zadania są zawsze przypisywane do niego, niezależnie od wybranego projektu.
- Pole color jest opcjonalne dla kompatybilności wstecznej
