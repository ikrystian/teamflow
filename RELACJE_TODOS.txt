=== PROBLEM ===
Dane z listy todo nie dodają się do bazy danych

=== ANALIZA PROBLEMU ===
Po analizie kodu znaleziono następujące problemy:

1. W API endpoint /api/tasks (GET) brakowało `todos: true` w include
   - Todos nie były pobierane razem z zadaniami z bazy danych
   - Poprawiono przez dodanie `todos: true` w include

2. W TaskDetailsDialog brakowało odświeżania danych
   - Dialog używał starych danych z props zamiast pobierać aktualne z API
   - Dodano pobieranie aktualnych danych zadania przy otwieraniu dialogu

3. W tasks-content.tsx brakowało onTaskUpdated dla TaskDetailsDialog
   - Po dodaniu/edycji todos lista zadań nie była odświeżana
   - Dodano przekazanie funkcji fetchTasks jako onTaskUpdated

=== STRUKTURA RELACJI ===

Task (zadanie)
├── todos: Todo[] (lista do zrobienia)
├── comments: Comment[] (komentarze)
├── subtasks: Subtask[] (podzadania)
├── timeEntries: TimeEntry[] (wpisy czasu)
├── images: TaskImage[] (obrazki)
├── project: Project (projekt)
├── assignee: User (przypisany użytkownik)
└── createdBy: User (twórca zadania)

Todo (element listy do zrobienia)
├── id: string
├── title: string
├── isCompleted: boolean
├── createdAt: DateTime
├── updatedAt: DateTime
├── taskId: string (klucz obcy do Task)
└── task: Task (relacja do zadania)

=== API ENDPOINTS ===

GET /api/tasks/{taskId}/todos
- Pobiera wszystkie todos dla zadania
- Sortuje po createdAt ASC

POST /api/tasks/{taskId}/todos
- Tworzy nowy todo
- Wymaga: title (string)
- Zwraca: utworzony todo

PATCH /api/tasks/{taskId}/todos/{todoId}
- Aktualizuje todo (title, isCompleted)
- Zwraca: zaktualizowany todo

DELETE /api/tasks/{taskId}/todos/{todoId}
- Usuwa todo
- Zwraca: potwierdzenie usunięcia
